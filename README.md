# 웹 개인정보 불법 유통 탐지 시스템

개인정보 불법 유통 사이트를 모니터링하여 KISA 신고를 위한 증거 수집 도구

## 프로젝트 목적

이 시스템은 웹상에서 개인정보가 불법적으로 거래되는 것을 탐지하고 모니터링하여 자료를 수집하는 방어적 보안 도구입니다.

## 주요 기능

- **키워드 기반 탐지**: 개인정보 불법 거래 관련 키워드 자동 탐지
- **웹 크롤링**: 게시판 및 개별 페이지 자동 분석
- **증거 수집**: 탐지된 불법 거래 정보를 엑셀 파일로 저장
- **중복 제거**: 동일한 URL이나 컨텐츠의 중복 저장 방지
- **실시간 모니터링**: 지속적인 웹사이트 모니터링 가능

## 🛠️ 시스템 구조

```
project/
├── main.py                    # 메인 실행 파일
├── config/                    # 설정 파일
│   ├── illegal_keywords.json  # 불법 키워드 정의
│   └── settings.py            # 시스템 설정
├── crawler/                   # 웹 크롤링 모듈
│   ├── manual_crawler.py      # 수동 크롤러
│   ├── rate_limiter.py        # 요청 제한 관리
│   └── constants.py           # 크롤링 상수
├── detector/                  # 키워드 탐지 모듈
│   └── keyword_matcher.py     # 키워드 매칭 로직
├── database/                  # 데이터베이스 관리
│   └── connection.py          # DB 연결 및 관리
├── results/                   # 결과 파일
│   └── crawl_results.xlsx     # 탐지 결과 엑셀 파일
└── logs/                      # 로그 파일
    └── app.log               # 시스템 로그
```

## 📦 설치 및 설정

### 1. 의존성 설치

```bash
pip install -r requirements.txt
```

### 2. Playwright 브라우저 설치

```bash
playwright install
```

### 3. 실행

```bash
python main.py
```

##  사용법

### 1. 개별 페이지 분석
- 특정 URL의 개인정보 불법 거래 여부를 분석
- 제목과 본문에서 키워드 탐지

### 2. 게시판 전체 분석
- 게시판의 여러 페이지를 자동으로 분석
- 각 게시글에서 불법 거래 키워드 탐지
- 탐지된 내용을 자동으로 엑셀 파일에 저장

### 3. 결과 확인
- 파일에서 탐지 결과 확인

## 🔍 탐지 키워드 분류

### 개인정보 DB 거래
- 부결DB, 완콜DB, 전날DB, 실시간DB 등
- 개인정보가 포함된 데이터베이스 불법 거래

### 계정 거래
- 네이버/다음 아이디 판매

### 금융 계좌 거래
- 대포통장, 법인통장 판매
- 금융 사기에 사용될 수 있는 계좌 거래
